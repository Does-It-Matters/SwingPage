# SwingPage
개발자가 프레임워크에 페이지와 이벤트를 전달해주면 화면 전환은 프레임워크가 해결한다.

## 프레임워크 구조 
- Event -> Router -> View 교체   

## 프레임워크 역할 
- 화면 전환 제어 흐름 분리 
- 뒤로 가기, 앞으로 가기, 다크 모드 등의 유틸리티 제공 

## 프레임워크 도입 시 장점  
- 개발자 경험(UX for dev) 향상
- 코틀린 DSL를 활용한 선언적 화면 구성 

## 모듈 설명 
- framework 모듈: 프레임워크 모듈 
- dev-intent 모듈: 개발자가 정의하는 이벤트 모음
- dev-page 모듈: 개발자가 정의하는 페이지 모음
